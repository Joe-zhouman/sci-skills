## Ethics & Regulatory（医学 / 生物 / 临床类必填，材料 / 工程类跳过）

> 以下声明按你的研究类型按需取用。用不上的删掉整节即可。每个声明一个代码块。
> 每个声明的结构：先确认是哪种研究类型，再给出标准措辞。把 `[placeholders]` 填好就能贴。

### 你的研究涉及以下哪些？（问用户，打钩）

- [ ] 人类受试者 / 人体样本
- [ ] 动物实验
- [ ] 临床试验（已注册）
- [ ] 细胞系
- [ ] 可能被误用的病原体 / 毒素 / 基因编辑
- [ ] 以上均不涉及 → 跳过本节

### 人类受试者 / 人体样本

> 必须提供：伦理委员会名称 + 批准号 + 日期。如果不需要审批（比如用的是公开数据库的去标识化数据），写豁免声明。

```
This study was approved by the [Full Name of Ethics Committee / Institutional Review Board] of [Institution] (Approval No. XXX, Date: YYYY-MM-DD). All procedures involving human participants complied with the Declaration of Helsinki (2024 revision).
```

### 知情同意

> 涉及可识别的个人数据、照片、病例报告时必须。

```
Written informed consent was obtained from all individual participants included in the study. For participants under 18 years of age, written informed consent was obtained from a parent or legal guardian.
```

### 临床试验注册

> 必须在招募第一个受试者之前注册。写注册机构和编号。

```
This clinical trial was prospectively registered at [ClinicalTrials.gov / WHO ICTRP / ChiCTR / EU Clinical Trials Register] under registration number [XXX] on [YYYY-MM-DD].
```

### 动物实验

> 必须提供：IACUC 名称 + 批准号 + 遵循的指南（ARRIVE 2.0 或 NIH Guide）。

```
All animal procedures were approved by the [Institution] Animal Care and Use Committee (Approval No. XXX, Date: YYYY-MM-DD) and conducted in accordance with the ARRIVE 2.0 guidelines and the [NIH Guide for the Care and Use of Laboratory Animals / national equivalent]. [Species, strain, sex, age, and number of animals] were used. [Anesthesia agent, dose, route] was administered. Euthanasia was performed by [method].
```

### 细胞系认证

> 涉及人源细胞系时必须。STR 谱 + 支原体检测 + RRID。

```
All human cell lines used in this study have been authenticated by short tandem repeat (STR) profiling within the last three years. Cell line identities were verified against the [ATCC / DSMZ / JCRB / Cellosaurus] reference database. All experiments were performed with mycoplasma-free cells. Cell line RRIDs are listed in the Key Resources Table.
```

### 生物安全 / 双重用途研究

> 涉及 Select Agents、增强病原体特性、基因驱动等情况时必须。写 IBC 批准 + 防护等级。

```
This research was approved by the [Institution] Institutional Biosafety Committee (Approval No. XXX). All experiments involving [agent / organism] were conducted at Biosafety Level [2 / 3 / 4] in accordance with [national / WHO biosafety guidelines]. The authors have considered dual-use implications and confirm that the benefits of publication outweigh potential risks.
```

### 上报指南清单（投稿时附加文件，不是 declaration）

> 以下不是粘贴到投稿系统文本框的东西，而是作为附加文件提交的 checklist。告诉用户哪个适用：

| 研究类型 | 指南 | 对应 Checklist |
|---|---|---|
| 随机对照试验 | CONSORT | CONSORT checklist |
| 系统综述 / Meta | PRISMA | PRISMA checklist |
| 观察性研究 | STROBE | STROBE checklist |
| 病例报告 | CARE | CARE checklist |
| 动物实验 | ARRIVE 2.0 | ARRIVE checklist |
| 诊断准确性研究 | STARD | STARD checklist |
| 质性研究 | SRQR / COREQ | — |

**给 agent**：先看稿件是什么类型，如果适用上述任一指南，提醒用户："你这种研究类型通常需要在投稿时附上 [XXX] checklist 作为附加文件。你的论文里已经有了吗？"

## Excluded Reviewers

> 回避审稿人需要你去找——不是问用户"你有没有要回避的"，而是读了手稿内容之后，反向推：谁跟这个工作有直接竞争关系？谁跟作者的导师/合作者有已知矛盾？然后向用户确认。

## Cover Letter Cheat Sheet（写封面信前填，不直接粘——AI 会按 writing-tips 帮你写成正式文段，写完给你确认）

> 这一节不是让你写大段英文。用关键词、短语、中英文混着来都行。把你知道的、想放进去的点列下来。AI 负责把它们变成学术英语。

### 背景钩子——"为什么是现在"的 2-3 个理由

```
[趋势 A 和趋势 B 最近交汇了，但没人把 A 和 B 放在一起做]
[这个方向去年刚出了一个关键突破——XXX 发了篇 Nature，但现在做的人还不多]
[我们正好有现成的数据和工具，卡位窗口还在]
```

### 核心发现——编辑读完能记住的那一个东西（一段话的量，列要点就行）

```
[最重要的那个发现——用中文写或者关键词都行]
[定量也列在这，AI 会转成定性表述——比如 "差两倍" 而不是 "p=0.003"]
[方法的独特之处——一句顶过]
[跟别人最大的不同在哪]
```

### 投稿后补充（写完封面信再填，存档用，写封面信时不看）

```
# 期刊契合度引用的论文（每投一个刊存一次，下次改投换刊时参考）
- [Author] et al. ([Year]), [Journal Name], DOI: [...]: [一句话——你的工作怎么接上这条线]
- ...

# 上次被拒反馈（如有）
[编辑说什么了——scope? 水平? novelty?]
```
