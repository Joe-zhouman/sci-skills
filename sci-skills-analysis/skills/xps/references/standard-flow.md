# 标准流程：保守参数下的可复现拟合

当用户说"就正常处理"时，用这份策略。目标是**任何一个合格的 XPS 分析师用同样的数据 + 同样的脚本都能复现你的结果**——不是"最好看"的拟合，是"最经得起审稿人挑刺"的拟合。

> **加载时机**：Step 0 判断用户走的是标准流程（而非叙事路径）时读。日常操作中跟 `fitting-strategy.md` 互补——那份教你怎么调，这份教你把每个旋钮拧到什么位置。

---

## 核心原则

1. **最小干预**：每一步选择最保守的参数。能不改就不改，实在要改就记录为什么改。
2. **灵敏度透明**：每个有自由度的选择（基线、RSF、峰数），至少两种合理方案都试一遍，报告差异。
3. **数据说话**：拟合器自己找到的值优先于人指定的值。约束只设物理范围，不设"想要的范围"。
4. **可复现为王**：任何选择都要能跟审稿人解释"为什么是它"。如果找不出理由 → 用默认值。

---

## 各旋钮默认策略

### 校准：内标 > C 1s > 不校

1. 样品含已知标准物（Au, Ag, Cu）→ **内标校准**。这是唯一"客观"的校准方式。
2. 没有内标 → **C 1s @ 284.8 eV**。虽然不准（Gengenbach 2022 证明漂移 ±0.5 eV），但它是领域标准——审稿人不会因为用 C 1s 而挑战你，只会因为用的是 C 1s 却声称"精确"而挑战你。
3. report 里声明："使用 C 1s 外来碳 284.8 eV 校准，该方法存在 ±0.5 eV 不确定性（Gengenbach 2022），优先使用内标校准"。

### 基线：Shirley 默认，Linear 对照

1. **默认 Shirley**——它是 XPS 领域的通用选择，审稿人看到"Shirley background"不会眨眼睛。
2. **同时试 Linear**——跑一次 `-m linear` 做灵敏度对照。如果面积比例差异 < 10%，在 report 里写"基线方法对结果不敏感"。差异 > 10%，报告两种结果。
3. Tougaard 仅在审稿人要求或在宽能量范围定量时才用——大部分情况 Shirley 够了。
4. 端点按目视平坦区域选。记录端点 BE 值。不确定选哪时偏宽不偏窄（宽端点漏掉一点非弹性散射电子，比窄端点切掉峰好）。

### 平滑：先不滑

1. **默认不滑。** Savitzky-Golay 平滑改变原始数据，审稿人会问"你滤掉了什么"。
2. 只有信噪比差到峰不可辨时才考虑。用最小窗口（w=3 或 5），在 report 里声明平滑方法和窗口大小。
3. 拟合后在残差图上检查——残差有正弦状振荡 ← 平滑过度，减窗口。

### 峰数：最少原则

这是标准路径和叙事路径分歧最大的地方：

1. **一个宽峰能覆盖的，不拆成两个。** 除非残差明确显示在该位置缺峰（单一正残差峰），否则不加。
2. **NIST 有记录的化学态才拆。** 凭"可能有"加一个峰 = 猜测，标准路径不猜。
3. **自旋-轨道分裂必须成对设。** p/d/f 轨道不配双峰 = 物理上错，不管是否"标准"。
4. 加一个峰 R² 从 0.88 跳到 0.96 但该峰无 NIST 依据 → 不加。在 report 里说"R² = 0.88，残留结构可能是化学态连续分散或仪器展宽，未强行分峰"。
5. **每增加一个峰，写一句理由**（写进 report 的方法部分）——"因 Si 2p 区域 ~103 eV 处残差存在正峰结构，加了一个 SiOₓ 峰（NIST 参考：SiO₂ 2p ≈ 103.4 eV）"。

### 约束：物理范围优先

1. **FWHM 范围**：0.5–3.0 eV。下限是实验室 XPS 仪器展宽（~0.3–0.5 eV），上限是"还算是单一化学态的合理宽度"。过渡金属氧化物放宽到 5 eV。
2. **峰位范围**：NIST 均值 ± 1.0 eV。松到拟合器有足够空间自己找最优。**不把范围收窄到某个具体 BE 值**——那是叙事路径的做法。
3. **amplitude > 0**。不需要上限。
4. 拟合后检查：有参数顶到边界的 → 放宽重跑。峰位落在 center_range 的 0.1 eV 边缘但 NIST 支持的范围是 0.5 eV → 约束太紧，放宽。

### 自旋-轨道双峰

1. 分裂值和面积比用 `peak-fitting.md` 中的物理常数表——不允许拟合器自己找。
2. 两个分量的 FWHM 必须相同（`sigma_range` 设成一样）。

### RSF：两套都报

1. **先跑 Scofield**（`--rsf-source scofield`）。这是理论截面，不依赖仪器，普适可复现。
2. **如果有用户仪器 RSF，也跑一版**。两套结果一起列在 report 里。
3. 差异 < 5% → 随便用哪个，注明用的哪个。
4. 差异 > 10% → 两套都报，声明"定量结果对 RSF 选择敏感，范围 X–Y atom%"。
5. IMFP 修正不做（标准流程的默认精度不要求到那个级别），在 report 里注明"未做 IMFP 修正"。

### 峰函数：pseudo_voigt 就够了

1. **默认 pseudo_voigt**。它比纯 Gaussian 物理（真实 XPS 峰总有一点 Lorentzian 成分），比纯 Voigt 快，R² 通常最好。审稿人不会挑战 pseudo_voigt。
2. 残差在峰两翼不对称 → 换 fraction 或换 Voigt 对比。如果 FWHM 绝对值对论文重要（比如讨论化学态分散程度）→ 换 Voigt 把 Lorentzian 和 Gaussian 贡献分开。

## 迭代终止条件

标准路径用跟叙事路径一样的收敛标准（见 `fitting-strategy.md` 的"何时停止"），外加一条标准路径特有规则：

**调完所有旋钮的保守默认值后，只调参数的物理范围（放宽约束、调整基线端点），不改决策（不加峰、不换基线方法、不换校准）。** 如果保守默认值的拟合已经满足收敛条件 → 停止。如果 R² < 0.80 或残差有系统结构 → 按 `fitting-strategy.md` 的决策树诊断，但每一步优先选最保守的行动。

## 报告：跟叙事路径的区别

标准路径的 report 模板同 `report-template.md`，差异：

- §1 方法部分多一句话："所有参数使用最保守默认值；基线灵敏度已检查；峰数由残差最小原则确定"
- §2 每个区域的解读从"结论"变为"观察"——说数据显示了什么，不说这个显示意味着什么
- §4 局限性额外声明："本报告基于最保守参数选择。如选择更宽松的约束或不同的基线方法，结果可能在一定范围内变化（具体灵敏度数值见各节）"
- §5 结论从"证明 X"改为"呈现 X"——如果用户没有具体 claim，这里写"数据能够说明..."和"数据不能说明..."

---

## 跟叙事路径的交叉

标准路径不是孤岛。用户可能在标准路径下跑完、出了 report、看图之后说"诶，我觉得 Si₃N₄ 确实是主相"——这时再切到叙事路径，用已有的 fit.json 做 baseline，只调峰数/约束/RSF 去强化这个结论。反之亦然：叙事路径调参调不动了，退一步用标准路径的保守参数看看"在这个设定下数据实际支持什么"。

这意味着**两条路径的 JSON 文件格式完全一样**——同一条 scripts 链，同一个 `state.json`。差异只在 agent 的决策逻辑。
