# 拟合微调策略

正文第二阶段给了"调一步 → 跑脚本看效果 → 不满意回去改"的循环，但没有说**先调哪个、怎么判断调对了、什么时候停**。这份文件补那个缺口——给 agent 一个结构化的决策框架，替代 XPS 老手的直觉。

> 这份文件在迭代卡住时读。不用从头读到尾——按你遇到的问题跳到对应章节。

---

## 旋钮影响力排名

XPS 拟合有五个可调维度。按对**叙事结论**（峰位归属、相对比例）的影响力从大到小排列：

| 排名 | 旋钮 | 影响力 | 调什么 |
|---|---|---|---|
| 1 | **峰数** | 毁灭性 | 多加一个峰 → 凭空多一个"化学态"。减一个峰 → 抹掉一个真实组分。这是最强的叙事旋钮，也是最容易滥用的。 |
| 2 | **基线端点** | 极大 | 端点外扩 → 所有峰面积同比例增大 → 原子% 全部变化。峰面积决定"谁多谁少"，直接影响 claim。 |
| 3 | **校准偏移** | 很大 | 整体平移 0.3 eV → 所有峰同时移 → 峰位归属可能全变。Si 2p @ 101.8 = Si₃N₄，偏 0.3 eV 到 102.1 = 说不清是什么。 |
| 4 | **峰的 center_range 约束** | 大 | 锁死峰位 → 峰被迫落在你指定的化学态位置。放松 → 拟合器自己找，可能偏到另一个化学态。 |
| 5 | **RSF 选择** | 中 | 同一组峰面积，换一套 RSF → 原子% 能差 10-20%。仪器厂家 RSF vs Scofield 理论值——哪个跟其他表征（EDS/XRD）更自洽选哪个。最后调。 |
| 6 | **峰函数 + FWHM 约束** | 中-小 | 换函数主要影响残差形状和 FWHM 绝对值，对峰位和面积的影响远小于前四项。 |

**调参铁律：从影响力最小的开始调。** 峰函数/FWHM 不够用了才动约束，约束不够用了才考虑峰数。绝不要一上来就加峰——那是结论先行，不是拟合。

## 残差读数法

残差图不是"好不好看"——它是诊断工具。`plot_region.py` 生成的图底部有残差线。看它的**形状**，不是大小：

| 残差模式 | 诊断 | 行动 |
|---|---|---|
| 随机白噪声，均匀分布在零两侧 | ✓ 拟合良好，没有缺峰 | 不需要动 |
| 某个位置有单一正峰（数据高于拟合） | 缺了一个峰——数据在那个位置有信号，拟合没抓到 | 在那个位置加峰（查 NIST 确认有化学态依据） |
| 某个位置有单一负峰（数据低于拟合） | 多了一个峰——拟合在那个位置塞了不存在的组分 | 去掉那个峰，或大幅缩小其 amplitude_range |
| 残差整体 S 形弯曲 | 基线不对——Shirley 的 S 形没跟上数据真实背景形状 | 换基线方法或调整端点 |
| 残差在峰的两翼不对称（一侧正一侧负） | 峰形不对——Gaussian/Lorentzian 比例不匹配 | 换 pseudo_voigt 的 fraction，或换 voigt |
| 残差在低 BE 侧系统偏高 | 基线端点太靠内，高 BE 侧的尾巴没减干净 | 基线低 BE 端点外扩 |
| 整个区域残差朝一个方向倾斜 | 基线端点不对称——一端选在信号区了 | 两端都移到平坦区域 |
| 残差振荡有规律（正弦状） | 平滑过度——Savitzky-Golay 窗口太大，把真实信号也滤了 | 减小平滑窗口或跳过平滑 |
| 两个相邻峰之间的谷底，残差有结构 | 两个峰的 FWHM 约束不匹配——一个太宽吞了另一个 | 收紧 sigma_range，让两个峰宽度可比 |

**判断优先级：先看残差形状，再看 R²。** R² = 0.95 但有系统 S 形残差 → 拟合有问题，不要被 R² 骗了。R² = 0.85 但残差完全随机 → 可能只是噪声大，拟合本身是对的。

## 迭代决策树

每一轮迭代的标准回路：

```
出图 (plot_region.py)
  ↓
看残差形状 → 有系统结构？
  ├─ 是 → 按残差读数法诊断 → 调对应旋钮 → 重新拟合 → 回到出图
  └─ 否（残差随机）
       ↓
     看 R²
       ├─ R² ≥ 0.95 → 看峰位归属和面积比例 → 支撑 claim？
       │    ├─ 是 → 进入"何时停止"检查
       │    └─ 否 → 换基线重做（端点/方法）→ 看面积比例是否反转
       │              ├─ 反转了 → 基线敏感，在 report 里声明
       │              └─ 没反转 → 接受当前结果，如实报告
       ├─ 0.80 ≤ R² < 0.95 → 可接受但不够好
       │    └─ 尝试：放宽约束 → 换峰函数 → 换基线方法
       │         任一步 R² 到 0.95+ 且残差随机 → 进入"何时停止"
       │         三步都试过仍 < 0.95 → 接受，在 report 里声明 R²
       └─ R² < 0.80 → 有根本性问题
            └─ 尝试：减少峰数（过拟合常见原因）→ 放宽所有约束 → 检查数据质量
                 仍 < 0.80 → 跟用户说明数据质量可能不足以支撑可靠拟合
```

**关键原则：R² 不是唯一标准。** 一个 R²=0.99 但残差有系统结构的拟合，不如 R²=0.92 但残差完全随机的拟合。后者只是噪声大，前者是模型错。

## 何时停止

满足以下所有条件 = 拟合完成：

1. **残差随机**：没有可辨识的系统结构（允许噪声幅度的随机波动）
2. **R² ≥ 0.90**：或 R² < 0.90 但残差随机且用户确认数据噪声大
3. **峰位归属有 NIST 依据**：每个峰的 center 落在至少一个 NIST 记录的 BE ± 1 eV 范围内（用 `lookup_be.py` 验证）
4. **FWHM 物理合理**：0.5–3.0 eV，且同一样品内不同化学态的 FWHM 差异 < 1.5 eV（排除过渡金属氧化物/稀土的宽峰）
5. **面积比例不自相矛盾**：claim 说"A 是主相"→ A 的面积确实最大；说"A 被消耗"→ A 的面积确实比对比样小
6. **多区域一致**：如果有多个区域（如 Si 2p + N 1s），同一化合物的 BE 在两个区域各自合理且互不矛盾

有一条不满足 → 继续迭代。六条都满足 → 停止，进入第三阶段收尾。

**停止不等于"完美"。** 真实 XPS 数据有噪声、有仪器展宽、有化学态连续分散——拟合永远是对真实谱的近似。目标是"诚实的、可辩护的近似"，不是"完美的拟合"。

## 过度拟合警戒线

多加一个峰 = R² 一定上升。但 R² 上升 ≠ 模型变好。以下任一触发 = 过度拟合，减峰：

- **无 NIST 依据的峰**：某个峰 center 落在任何 NIST 记录 ±1 eV 范围之外 → 这个峰没有化学态对应，是拟合器"捏造"的
- **FWHM < 0.5 eV**：实验室 XPS 仪器展宽下限 ~0.3–0.5 eV。FWHM 比这个还小 → 峰是拟合器硬挤出来的，物理上不存在
- **amplitude 极小**（< 主峰 amplitude 的 5%）：这个峰贡献微乎其微，加不加对包络线几乎没影响。除非 claim 就需要证明"微量组分存在"
- **两个峰 center 间距 < 0.3 eV 且 FWHM 都 > 1.5 eV**：两个宽峰中心几乎重叠 → 拟合器无法区分它们，stderr 会很大。合并为一个峰
- **每多加一个峰，相邻峰的 FWHM 剧烈变化**（> 50%）：新峰在"偷"相邻峰的面积和宽度，说明这些峰本就不该拆开

**铁律：峰的化学意义 > 峰的拟合优度。** 多一个 NIST 无记录的峰去提高 R² = 造假。宁可 R² 低 0.02，不要多一个没有化学意义的峰。

## 跨样品 / 跨区域一致性

### 对比样品（before/after, treated/untreated）

same claim（"改性后 Si₃N₄ 增加"）需要对比两个样品时：

1. **相同的校准方法**：两个样品用同一种校准（同内标或同 C 1s 偏移量），否则峰位不可比
2. **相同的基线方法 + 端点尽量一致**：基线方法必须相同（HARD-GATE 已规定）。端点 BE 值也尽量一致——端点不同 → 面积基准不同 → 比例不可比
3. **相同的峰数和峰位约束**：两个样品用同样的 peaks.json 模板（label 和 center_range 一致），只改 amplitude 初始值。峰数不同 = 基础不一致
4. **对比图用相同的 y 轴 scale**：`plot_compare.py` 默认自动 scale，但两个样品信号强度差异大时，人工指定 y 轴范围让视觉可比

### 同一元素的不同轨道（如 Si 2p + Si 2s）

如果同一元素的两个轨道都在数据中：

1. **校准偏移必须一致**：同一个样品不会有两个不同的荷电量
2. **化学态归属必须一致**：Si 2p 说有 Si₃N₄，Si 2s 不能说没有——除非 Si 2s 区域信噪比太低无法拟合
3. **定量比例应相近**：同一化合物的两个轨道算出的原子% 应接近（允许 ±15% 差异，来源：RSF 不同 + 探测深度不同）

## 诚实边界：当数据不支撑 claim 时

本 skill 的 HARD-GATE 说"先有结论，再用合理的 XPS 数据处理去佐证它"。但"合理的"有边界：

### 可以做的（合法自由度）

- 在 NIST 支持的 BE 范围内选择校准偏移量
- 选择基线方法和端点（记录在案，做过对比）
- 用 NIST 数据支持的化学态去指认峰
- 在物理合理范围内设 FWHM 约束
- 做灵敏度分析：换基线 → 看结论是否稳健 → 如实报告

### 不能做的（越界）

- 校准偏到目标化学态 NIST 范围之外（e.g. Si₃N₄ NIST 范围 101.2–102.4，你偏到 102.8 还说是 Si₃N₄）
- 为撑面积多加一个没有 NIST 依据的峰
- FWHM 设到物理上不可能的值（< 0.3 eV 或 > 5 eV）
- 基线端点选在明显还有信号的位置
- 只报支持 claim 的结果，隐瞒不支持的结果

### 当数据真的不支撑 claim

如果调了所有合法旋钮、做了基线灵敏度分析、查了 NIST 确认峰位归属，结论仍然不支撑 claim：

1. **如实告诉用户**：哪些具体指标不支撑（峰面积比例？峰位归属？），列出你调过的参数和范围
2. **给替代解读**：数据实际支持什么结论？有没有和 claim 不矛盾但更保守的表述？
3. **建议补充证据**：什么其他表征能帮上忙？（XRD 确认物相、TEM 看形貌、EDS 给元素比）
4. **把调参历史写进 report**：你在 §4 "局限性" 里声明"尝试了 X/Y/Z 基线方法和 A/B/C 峰数，结论在合理参数范围内稳键/不稳健"

**记住：XPS 不是全能的。** 有些问题它回答不了——化学态太接近分辨不开、含量太低检测不到、样品荷电太严重。诚实地承认"这份数据能说的就这么多"比硬凑一个 claim 更专业。

## 速查：常见卡住场景

| 症状 | 最可能原因 | 先试这个 |
|---|---|---|
| R² 不错但残差 S 形 | 基线方法不匹配 | 换基线方法（Shirley → Linear 或反过来） |
| 两个峰总黏在一起 | center_range 太宽，两个峰 FWHM 都大 | 收紧 center_range，减小 sigma_range 上限 |
| 某峰 FWHM → 0 | 峰不存在 | 删除该峰 |
| 某峰 amplitude 顶到上限 | amplitude_range 上限太低 | 放宽上限 |
| 峰位偏到 center_range 边界 | 初始值离真实峰位太远，或约束跟 NIST 冲突 | 查 NIST 确认 BE，调整 center + center_range |
| 加一个峰 R² 从 0.88 跳到 0.96 | 这个峰可能是真实的 | 查 NIST 确认该位置有化学态依据 → 有 = 保留，无 = 过度拟合 |
| 基线端点附近残差大 | 端点选在信号区了 | 端点外扩到平坦区域 |
| 拟合结果对基线端点极敏感 | 基线扣除是这个谱的主要不确定性 | 做基线灵敏度分析，在 report 里定量报告影响 |
