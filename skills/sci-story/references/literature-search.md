# Literature Search — 引文搜索策略

Introduction 和 Discussion 的引用必须来自真实文献。优先学术搜索工具，退而求其次用通用搜索。不编文献、不空占位。

## 搜索工具优先级

写 Introduction 和 Discussion 需要引文献时，按以下顺序找工具：

1. **学术搜索 MCP**（如 `mcp__common__vps-searxng_search`）→ 直接搜学术关键词
2. **文献库 MCP**（如 `mcp__doc__context7-query-docs`）→ 查领域文档/论文库
3. **通用搜索**（`WebSearch`）→ 都不行时退到这里

搜到论文后，**验证期刊质量**：如果项目里有 `sci-submit/scripts/query-journal.py`（需要 EasyScholar API key），用它查分区/IF。没有就跳过验证，不要为此阻断流程。搜到的论文如果来自明显低质量期刊（掠夺性期刊、无同行评议、会议摘要冒充论文），不管分区如何——不引。

## 各层的引文要求

### Layer 1: 大背景（1-2 句）

job: 定调——"X 领域为什么重要"。要一个不可否认的 stake。

**引谁**：Q1 / 一区优先。至少三篇独立来源从不同角度合力支撑一个 claim——"AI 导致算力暴涨"不能只靠一篇综述，需要算力需求、功耗上升、硬件现状各一篇独立出处。

**搜什么**：`[领域关键词] review` 给方向；再用 `[领域关键词] [具体子方向]` 找独立支撑。

**验证**：有 `query-journal.py` 就跑一下。没有就凭期刊声誉判断——Nature/Science/PNAS/Cell/领域顶刊无需验证。

### Layer 2: 小背景 + 现状（1-3 段）

job: 当前 SOTA 是什么、卡在哪。归纳现有方法/理论的共同瓶颈。

**引谁**：Q1 / 一区优先。如果某个关键工作发表在 Q2 但高度相关——可以引，但不要以它为主。Review 和 original article 都可以。核心筛选标准：**这篇文献是否代表了当前最佳实践/最先进水平？** 如果不是，它不配出现在 bottleneck 论证里。

**搜什么**：`[具体子领域] [方法关键词] recent advances`、`[瓶颈关键词] limitation`

**验证**：同 Layer 1。

### Layer 3: Prior work（最相关的几个工作）

job: 和论文最直接相关的具体工作。公平对待——"Although X showed Y, they did not address Z."

**引谁**：这些是跟你正面对话的工作——审稿人如果是这个领域的一定认识它们。**质量不如相关性重要**——一篇高度相关且会被审稿人要求引的 Q2 论文，必须引。但要确保它是 peer-reviewed 的正经期刊，不是 arXiv 预印本或会议摘要冒充的。

**搜什么**：搜索范围收窄——`[具体技术/方法名] [你的问题域]`

**验证**：引之前读摘要确认真的跟你的 gap 相关。不要只看标题就引。

### Layer 4 & 5: Gap + Present study

这两层**不引新文献**。Gap 陈述是你自己的论证（从 Layer 2-3 推导出来）。Present study 是你的工作。

## 搜索纪律

1. **搜索完再写。** 不要在写的过程中"我猜有一篇文献说 X"——搜了再说。搜不到就标 `[DOI needed: 描述]`。
2. **读了摘要再引。** 引用的每篇文献都要确认其核心结论跟你要引的内容一致。只看标题引 = 等着审稿人挑出来。
3. **搜索结果不落入同质化陷阱。** 搜同一个 query 的多个搜索引擎可能返回相同结果——那是同一个证据源换了壳。一篇论文在你搜的四个数据库里都出现 ≠ 四篇独立论文支持你的论证。算一次。
4. **落盘为占位符**：`[DOI: 10.xxxx/xxxxx]`。最终 Zotero 插入是人的活。
5. **搜不到不强搜**：一个领域如果搜索结果少于 3 篇相关——可能是关键词错了，换个搜法。如果换了还是少——可能就是没多少工作——在 Introduction 里如实说"few studies have addressed X"，不要编。
