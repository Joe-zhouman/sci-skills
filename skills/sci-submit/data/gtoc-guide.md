# Graphical Abstract / TOC Figure

## ⚠️ AI 绘图的真实风险——不是"一定会被查"，是"说不准什么时候被查"

先说清楚。AI 绘图这事在学术界已经存在了好几年——有人用 Midjourney 生成的图投了 Science，有人用 DALL-E 画的图当了封面。**目前没有统一部署的投稿系统检测器。** 如果你实测能过 peer review，那短期内大概率还能继续过。

但有几个正在发生的变化需要注意：

**为什么以前没事、以后不一定**。早期的 AI 图是没有水印的（Midjourney 2023 年之前的版本、早期的 Stable Diffusion）。所以编辑的审稿经验是：AI 图要么有明显的视觉缺陷（手指不对、文字乱码），要么看不出就看不出了。但现在这两个最大的 AI 绘图工具——GPT-image-2（OpenAI DALL-E 3）和 Nano Banana（Google Gemini）——都在生成过程中嵌入 SynthID：一种频域水印，不是在像素上叠个半透明 logo，而是在图像扩散生成的每一步通过概率采样影响像素值，在 FFT 频谱上形成可检测的统计模式。你现在每次生成都在网上留痕。

**会检查吗？** Google 的 SynthID Detector 已经在 Gemini App 和 Google Lens 里公开可用，任何人上传图片都能查。出版社接不接、什么时候接，取决于法务部门什么时候意识到这是合规缺口。从技术上说，接一个 SynthID Detector API 比接 CrossRef 或者 iThenticate 还简单。

**版权问题才是硬刀。** 即使水印检测没触发，图是你用 AI 生成的这件事本身就有版权风险——美国版权局和中国版权局目前的共识是 AI 生成的内容不属于"人类创作"。如果你对 AI 生成的图做了充分的人工修改、让它成为论文整体原创论证的一部分，有可能争取版权。但纯由 AI 生成的图，你签不了正式的版权转让声明。

**实用建议**：AI 图可以用——当灵感参考，甚至当 PPT 的背景素材——但终稿溯源链条应该干净。用 PPT 重新画一遍，哪怕结构跟你 AI 生成的参考图一模一样，至少它是你亲手画出来的矢量文件，你可以签版权声明。

## 怎么做

用 **PPT 手绘**。这是最稳妥的方式：

- PPT 画的图是矢量的，导出成 TIFF 或 EPS 就是出版级的。
- 做一次，改投另一本期刊的时候改改尺寸和配色就能复用。不依赖任何 AI 工具，不用担心版权问题。
- 箭头、文本框、配色、对齐——PPT 的图形引擎优化了二十年，比你想的好用。
- Elsevier 甚至接受 PPTX 直接上传。

**工作流**：

1. **PPT 画好** → 按目标期刊要求导出（Elsevier: 1328×531px 300dpi TIFF）
2. **截图给我** → 我用识图工具帮你看：文字在缩略尺寸下能不能读、视觉流向清不清晰
3. **改了再截** → 一个人容易看不出毛病，多一双眼睛

## 通用设计规则——任何 GTOC 都适用

以下来自四张不同期刊的已发表 GTOC（`assets/gtoc1-4.png`）的共性分析。不管你怎么布局，这几条不变。

### 结构：从起点到终点的线

**从上到下、从左到右。** 这条线可以是一根箭头——也可以没有箭头，纯靠空间排布和标题层级来暗示。不论怎么实现，必须明确看得见一个起点和一个终点。

**终点放结论。** 结论是唯一用完整句子的位置，高对比度边框让它无法被忽视。看 `assets/gtoc1.png` 的橙边框结论框——这是四个例子里最直观的结构层级。

### 图标：两眼能认出来

选一眼能认出来的图标（材料晶格、箭头、齿轮、反应器、患者、药丸）。不要抽象图标——在 200px 高度下，抽象图标像污迹。**图标配短标签**，不要只放图标或只放文字。两者结合可以减少 50% 的文字量。

### 文字：能省就省

- **唯一需要完整句子的只有结论。** 其余全是短标签或数字。
- 图中定义所有缩写。审稿人和编辑不一定认识，即使你的目标读者应该认识。
- 关键数字用大号粗体突出显示——可以完全从 Highlights 里复用。
- 缩放测试：缩小到 5cm 高，标题和结论还能读吗？读不了就砍字。

### 用颜色——唯一的作用是分组

- **3-5 种颜色就够。** 一个高亮色，两三种板块背景色，一种正文色。无装饰色。
- **永远不要仅用颜色区分信息。** 位置和文字标签是主要区分方式。饱和度调到零，图必须还能用。

### 不能做的事

**不要贴 Figure 1 替代 GTOC。** 编辑一眼能看出来。单独做一张。

## 怎么做——两个省事的模板

> 以下纯属作者个人经验。很模式化，不需要设计意识，照搬就能用。不代表你一定要这样做。如果你的工作用这两个模板都不顺手，回到上面的通用规则自己想布局——那才是真正管用的东西。

PPT 打开，新建一页空白幻灯片。两个模板选一个。

### 模板一：并排四列

```
┌────────────────────────────────────────────┐
│                  Title                      │
├──────────┬──────────┬──────────┬───────────┤
│  Block 1 │  Block 2 │  Block 3 │  Block 4  │
└──────────┴──────────┴──────────┴───────────┘
```

标题占满宽 → 下面并排四个等宽格。适合流程少、每个块信息量均匀的情况。

### 模板二：田字格

```
┌────────────────────────────────────────────┐
│                  Title                      │
├──────────────────┬─────────────────────────┤
│    Block 1       │      Block 2            │
├──────────────────┼─────────────────────────┤
│    Block 3       │      Block 4            │
└──────────────────┴─────────────────────────┘
```

标题占满宽 → 下面 2×2 四格。适合关键图表比较大的情况——图表占一格（比如左上），不压缩。或者流程复杂的时候，两格拆成主流程+细节。

---

两个模板共用下面的填法。

### 标题栏

PPT 标题占位符拉满页面宽度。一句话研究问题，粗体。

### 四个格子填什么——三选一，按需搭配

三个内容块，四个格子搭。格子不够就合并，**绝不能空着**。

**A. 研究流程**。矩形框 + 箭头：`[Step 1] → [Step 2] → [Step 3] → [Step 4]`。每步短标签 + 一个小图标。不知道怎么把流程可视化？截个图给我。

**B. 关键图表**。从手稿挑 1-2 张最有代表性的。不需要跟论文里一样——截关键区域，去坐标轴数字和图注，配一句话。一张图配一句话，绝不多说。

**C. 核心发现 + 意义展望**。直接复用 Highlights，缩减到 2-3 条（每条看到 200px 高还能读吗——超过 60 个字符就砍），末尾挂一句话——"解决了什么问题，为谁提供了什么价值"。"意义展望"单独占一个格子太浪费了——挂在这就行。

### 怎么搭——四个格子随便拆

两个模板只是最省事的起点。四格可以拆成无数种玩法——这才是模板真正省事的地方：

```
并排四格:  ▯▯▯▯   可拆成  ▯▯▯▯   全部独立，三道线
                        ▯▯ ▯▯   二二对半分
                        ▯▯▯ ▯   三一分布
                        ▯ ▯▯ ▯   一三一布局
                        ▯ ▯▯▯    左边一格独立，右边三个竖排
                        ▮▮▮▮   全合并，只放一张大图

田字格:  ▯▯    可拆成  ▯▯   二二独立
         ▯▯            ▮▯   左合并 + 右独立
                        ▯▮   上独立 + 下合并
                        ▮▮   全合并
```

**不用纠结怎么拆。** 打开 PPT，拉四个矩形，觉得哪个格该大就往大的方向拖，觉得该小就缩。三块内容（A B C）往四格里塞，格子不够就合并——**但绝对不能空着。** 空一个格子等于在说"我不知道这里该放什么"，编辑的潜台词是"作者没用心"。四个格子要么全填满，要么合并掉。

### 加图标

不需要自己画。中文用户直接去 **[iconfont](https://www.iconfont.cn)**（阿里图标库，免费，SVG/PNG）。搜"实验""材料""电池""细胞""反应器"。免费的专门科研图标：**[Bioicons](https://bioicons.com)**（2800+ 生物/化学/分子，CC0/MIT）、**[Servier Medical Art](https://smart.servier.com)**（3000+ 医学插图）、**[SciDraw](https://scidraw.io)**。BioRender 和 Noun Project 不要用（水印 / 署名要求太麻烦）。

### 给我看

画完截个图。我帮你检查缩略尺寸能不能读、视觉流向对不对。

### 想再好看一点？

丐版模板够用了——对于大多数期刊，GTOC 的及格线是"编辑一眼看懂"，不是"漂亮"。但如果你投的期刊对 GTOC 比较看重（封面、highlight 推荐位），想做得更精致，可以试试 `front-design` 或 `kami` 这类前端设计 skill——它们能帮你搭更灵活的布局和控制更细的视觉细节。只是投入时间会多不少，投个三区四区完全没必要。

## 投稿时

- 文件类型选 **"Graphical Abstract"**（不是 "Figure"）
- 文件名：`graphical-abstract.tiff` / `toc-figure.eps`
- sci-submit 不替你画图——但会在 Workflow C 的对应页面提醒你上传

## 其他出版社

- **ACS**: TOC/Abstract Graphic，多彩色，比例常 8.25 cm × 4.75 cm（宽×高）
- **RSC**: TOC Graphic，通常 8.25 cm × 3.25 cm（宽×高），300 dpi，TIFF/EPS
- **Wiley**: 因刊而异
- **Springer Nature**: Nature 主刊一般不要求，部分子刊和 Communications 系列可能要

> 每种期刊规格不同——投稿前必查该刊 Guide for Authors。
