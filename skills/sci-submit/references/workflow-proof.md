# Workflow G: Proof Review

## 这是什么

接收之后，期刊排版完，通知你进系统校稿。**不是发 PDF 给你，是让你登录投稿系统（Proof Central、e.Proofing 之类）在线逐条确认。** 24-48 小时窗口。

**这不是让你改内容的窗口——是你最后一道防线。** 过了这道线，所有错误都永久印刷。

## 前置准备

打开两个东西，左右分屏：
- **左边**：期刊的在线 proof 系统
- **右边**：`sci-submit/manuscript-meta.md`

## 能做和不能做的

| 能做 ✅ | 不能做 ❌ |
|---|---|
| 纠正拼写错误、语法错误 | 加新的数据、新的分析 |
| 纠正作者姓名/单位/基金号 | 换图、加图、删图 |
| 纠正图注、表中数字（排版引入的） | 大段改结论 |
| 调整参考文献格式错误 | 调整作者顺序 |
| 回答排版编辑的 query | 改稿件类型 |

**作者名单已锁死。** 加人、删人、调顺序——proof 阶段编辑几乎不会同意。少数期刊允许通讯作者变更（必须写书面说明），其他改动基本没戏。这件事投之前就该解决：投稿前把名单发给导师确认，revision 阶段也别动。到了 proof 才想改，晚了。

**改动越少越好。** 每次改都会触发新一轮校对，可能影响发表日期。只改必须改的。

## 五步校对——顺序不能乱

### 第一步：编辑的改动和 Query

登录在线 proof 系统，第一屏是编辑已经改过的标记和 Query 列表。**先处理完这些再碰正文。**

**改动**：编辑的拼写/语法改动——对的接受、错的拒绝（写 "Please retain original: [原文]"）。格式/标点——几乎都是对的，编辑按期刊 house style 调的，接受。单位、基金号、参考文献——排版时最容易被改坏的三个点。对照 `manuscript-meta.md` 核查。不回复 = 默认接受，以后发现错误编辑会说 "Proof 阶段你为什么没提"，责任归你。

**Query**：每个都回，**没问题的也要回**。"Please confirm that the title, author list and funding information are correct"——你核对了，一个字没错，回 "Confirmed"，不要留空。所有确认框都点完了才叫 proof 已提交。留一个没点，过了 deadline 你的 proof 就被视为未提交，编辑会发邮件催你，这次投稿的收尾阶段又多拖几天。

### 怎么跟编辑说话

**编辑不是机器，也不是你的对手。在这个阶段，你们的目的完全一致：把这篇文章正确、干净地印出来。** 他不是审稿人——他不会挑你内容的毛病。他是排版员，他需要的是清晰的指令。

**说事，不说套话。** 不需要 "Dear Editor, thank you for your careful work on our manuscript, we greatly appreciate..."——proof 系统里的批注框就这么大，你打套话的时间够你多核对两个基金号了。事无巨细都可以说，但每一句话都指向一个具体操作。

**举例**：一篇论文的组图编号是 a, b, c, d, f, g, h——中间缺了 e。不用写 "We noticed that the figure labels contain a numbering error. Specifically, panel e is missing and subsequent panels are mislabeled. Please change panel f to e, g to f, h to g."

直接批注：`f → e, g → f, h → g`。他看得懂。

### 第二步：作者信息

打开 `manuscript-meta.md`，逐人核对。**最容易被跳过、后果最严重。** 名字印错跟一辈子。

- [ ] 姓名拼写——尤其拼音（Zhang vs Chang、Zhou vs Zou）
- [ ] 作者顺序
- [ ] 通讯作者标记和邮箱——逐个字母读
- [ ] ORCID 数字、单位编号

### 第三步：基金信息

- [ ] 每个基金号——一个数字不差
- [ ] 每个资助机构全称——拼写完整

基金号错了，结题可能不被承认。

### 第四步：图表、参考文献、正文

**图表**：分辨率够不够、文字有没有被截断、表格有没有乱行。图变糊了 → 在线系统里换 300dpi 原图。

**参考文献**：同一作者没出现两种格式、"J. Smith" 和 "Smith, J." 不能混用。DOI 没丢。卷期页码没被截断。

**正文**：有时间从头读——最好。没有的话（24 小时窗口很常见），至少读标题、摘要、每个节的标题。**读正文的技巧：从最后一页往前读**，打破正常阅读节奏，眼睛不容易跳过拼写错误。

### 第五步：朗读检查

**眼睛会骗你，耳朵不会。** 大脑会把你熟悉的句子自动补全，跳过拼写错误。耳朵没有自动补全功能。

Edge 右键 "Read aloud"，Chrome 装个免费 TTS 插件，Linux 用 `spd-say`。默认语速太快——调到能跟读的节奏，一般 0.5×-0.75× 刚好。音质烂没关系——你不是在欣赏有声书，是在抓错别字。让机器把正文从头到尾念一遍，你一边听一边盯着在线 proof 页面——**听觉和视觉双重校对，一个人顶两个人的眼力。**

## 提交前

在线系统提交之前，确认：
- [ ] 每条编辑改动都回复了
- [ ] 每个 query 都回答了
- [ ] 作者信息和基金号都核对过了

提交后很多系统**不能撤回**——确认之前不要点 Submit。

## 校完后

1. **截图留底**——在线系统通常不给你一个带批注的 PDF 存档，截几张关键改动页
2. 更新 `submit-history.md`：

```markdown
### YYYY-MM-DD — Proof reviewed for [Journal Name]
- MS#: [ID]
- Corrections: [简要列改了什么，几条]
- Queries answered: [N] 条
- Status: Proof submitted
```

3. 告诉用户：

> "Proof 回完了。从今天开始不要再动这篇文章的任何内容——已经在出版流水线上了。等着上线通知就行。"

## 时间管理

**24-48 小时是常见的 proof deadline。** 收到通知立刻打开系统看一眼 deadline 是什么时候。如果时间太紧：回复编辑要求 24 小时延期，多数期刊会同意。
