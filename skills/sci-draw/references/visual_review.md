# 人眼复核清单

绘制 → 导出 → 人眼逐项核对。程序检查抓文件格式错误（DPI、尺寸），人眼抓感知问题（可读性、配色、对齐）。两项都过才算合格。

打开导出的 PNG 和 PDF，**逐条核对，不要扫一眼说"看起来不错"**。

## 检查清单

### 1. 乱码 / 方框

- 中文有没有变成 □□□ 方框 / 豆腐块？
- 负号、`±`、`×`、`μ`、`Δ`、希腊字母、上下标有没有缺字？

### 2. 字号可读

- PDF 100% 比例下，轴标签、刻度数字、图例能否轻松阅读？
- 任何文字 < 6 pt？→ 投稿检查直接拒。

### 3. 文字裁切 / 重叠

- 标题、轴标签、图例、标注有没有被画布边缘裁掉？
- 图例有没有压住数据点、线、柱？
- 显著性标注、数值标签之间有没有互相叠？
- x 轴刻度标签有没有挤成一团？

### 4. 子图编号对齐（多面板必查）

- a/b/c/d 标签是否同行等高、同列左缘对齐？
- 风格是否一致（不能有的 `a` 有的 `(a)`）？

### 5. 子图间距 / 互相侵入（多面板必查）

- 子图之间有没有重叠？y 轴标签有没有伸进左边邻居？
- colorbar 有没有和子图挤在一起？

### 6. 配色 / 灰度

- 各类别颜色能区分吗？有没有用到红绿对比？
- 打开 `_grayscale.png`，灰度下还能分开吗？

### 7. 数据完整性

- 有没有数据点 / 曲线 / 误差棒被坐标轴范围切掉？
- 最高的柱、最外的点、最宽的误差棒是否都在框内可见？

### 8. 跨子图一致性（多面板必查）

- 同一个变量在多个子图里是否同色、同标记、同量纲？
- 共享含义的坐标轴范围是否一致？

### 9. 论点传达

- 只看这张图，能不能理解它要论证什么（对照 Step 0 的 claim）？

### 10. Illustrator 级问题

- 文字微调、标签旋转、复合布局精修 —— 有些事代码磨一小时不如 Illustrator 两分钟。
- 记下来，导出 PDF 后手动改。别跟 matplotlib 死磕。

## 回改对应表

发现问题后回对应步骤改，不要手动 P 图。

| 发现 | 回改 |
|---|---|
| 中文/符号缺字方框 | Step 3: `setup_style(lang='zh')`；确认 `axes.unicode_minus=False` |
| 文字被裁切 | `layout_tools.finalize_figure(fig)`；标题过长换行/缩短 |
| 图例压住数据 | `ax.legend(loc=..., bbox_to_anchor=(1.02,1), frameon=False)` 移到图外 |
| 标注文字互相叠 | 调整偏移；减少标注数量（只标关键的） |
| x 轴刻度重叠 | `ax.tick_params(axis='x', rotation=30)`；减少刻度；缩短标签 |
| 子图标签不对齐 | `layout_tools.add_panel_labels(fig, style='nature')` |
| 子图互相侵入 | `finalize_figure(fig)` 或建图时确保 `constrained_layout=True` |
| 配色不可区分 / 灰度糊 | 换 Okabe-Ito / `colorblind` 调色板 + 加线型/marker 冗余编码 |
| 数据被切掉 | 放宽 `set_xlim/set_ylim`，或 `ax.margins(0.05)` |

## 循环纪律

- **每改一处就重渲一次** —— 不要一次改五处然后猜结果。
- **最多 3 轮**：3 轮还过不了，多半是图型选错了（回 Step 2 重选）或维度太多（拆图）。
- **留痕**：每轮发现的问题和改法记录在 description log 里，让用户知道图为什么是现在这样。
